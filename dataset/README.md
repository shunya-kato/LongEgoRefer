# State Captioning Benchmark — `longegorefer.json`

A benchmark dataset for **temporally and spatially grounding referring expressions that describe the state changes and interactions of a target object** within long egocentric videos (Ego4D / EgoTracks).

Each sample corresponds to a single interval (occurrence) in which an object is actively used (`is_active`), and carries a human-verified one-sentence referring expression (`caption`), the sequence of object bounding boxes within that interval (`bbox`), and the human–object interaction phrases extracted from the referring expression (`interaction`).

- File: `dataset/longegorefer.json`
- Number of samples: **1,498** occurrences
- Unique videos: 433 / Unique clips: 769 / Unique object titles: 442
- Source data: [Ego4D](https://ego4d-data.org/) full-length videos + [EgoTracks](https://github.com/EGO4D/episodic-memory) annotations

---

## Format

The top level is a **list** of occurrences (`list[dict]`). Each element has the following fields.

### Identifiers & metadata

| Field | Type | Description |
|---|---|---|
| `id` | str | Sample ID, in the form `{clip_uid}_{object_id}_{occurrence_index}`. |
| `video_uid` | str | UID of the source Ego4D video. |
| `clip_uid` | str | UID of the EgoTracks exported clip. |
| `object_id` | str | Object ID within the clip (the key in the EgoTracks query set). |
| `object_title` | str | Object name (e.g., `"plastic basin"`). |
| `occurrence_index` | int | Index within the object's `attributes_occurrence`. |
| `caption` | str | **The referring expression (evaluation target).** An object-centric description of state and interaction. Human-verified and corrected. Average 63.6 words (range 19–140). |
| `video_duration` | float | Length of the source Ego4D video in seconds. Average ≈ 2715 s (about 20–118 min). |

### Attribute flags (from EgoTracks annotations)

Boolean values describing the object's state during the occurrence interval.

| Field | Description | Distribution in this data |
|---|---|---|
| `is_active` | The object is actively manipulated / used | All `True` (filter condition) |
| `is_moving` | The object is moving | True: 1279 / False: 219 |
| `is_location_changed` | The object's location has changed | True: 1128 / False: 370 |
| `is_transformed` | The object's shape/state has been altered | True: 165 / False: 1333 |
| `is_recognizable` | The object is in a recognizable state | True: 1495 / False: 3 |

### Temporal boundaries (start/end of the occurrence)

The same interval is stored in three coordinate systems.

| Field | Coordinate system | FPS |
|---|---|---|
| `start_frame_number` / `end_frame_number` | Annotation frame number within the EgoTracks clip | 5 fps |
| `exported_clip_start_frame_number` / `exported_clip_end_frame_number` | Frame number within the EgoTracks exported clip | 30 fps |
| `video_start_frame_number` / `video_end_frame_number` | Frame number within the source Ego4D video | 30 fps |

The average occurrence length is about 28.7 s (range 10–443 s).
For temporal grounding, the ground-truth time span is computed in source-video coordinates as `video_start_frame_number / 30` to `video_end_frame_number / 30` (see `src/temporal_grounding_baseline.py`).

### Bounding boxes `bbox`

The object's position at each annotation frame within the occurrence interval (`list[dict]`).

| Key | Description |
|---|---|
| `frame_number` | Frame number within the EgoTracks clip (5 fps) |
| `video_frame_number` | Frame number within the source Ego4D video (30 fps) |
| `exported_clip_frame_number` | Frame number within the exported clip (30 fps) |
| `x`, `y`, `width`, `height` | Pixel-coordinate rectangle (top-left origin). For spatial IoU, convert to `[x, y, x+width, y+height]`. |
| `original_width`, `original_height` | Frame resolution. Multiple resolutions are mixed across videos (1440×1080 / 1920×1080 / 1920×1440 / 2560×1920, etc.). |
| `interpolated` | If `True`, the box was generated by interpolation |

### Interaction `interaction`

A list of verb phrases in the `caption` (referring expression) that **describe interactions the camera wearer performs on the target object** (`list[str]`). Extracted by `dataset/extract_interaction.py` using GPT, keeping the original words from the referring expression verbatim (without lemmatization). Interactions with objects other than the target are excluded.

| Item | Value |
|---|---|
| Type | `list[str]` (e.g., `["added", "agitated", "grasped", "slid"]`) |
| Non-empty samples | 1,305 / 1,498 (193 are empty lists) |
| Phrases per sample | 2.64 average over all samples / 3.03 average over non-empty samples (max 9) |

Example uses: performance analysis by presence/type of interaction, extracting samples that involve state changes, etc.

## Construction pipeline

Generated by the scripts under `dataset/`.

1. **Clip frame extraction** — `dataset/extract_ego4d_clip_frames.py`
   Expand EgoTracks clips into image frames.

2. **Automatic state-caption generation** — `dataset/generate_state_caption_dataset.py`
   Take intervals filtered from EgoTracks `attributes_occurrence` by flags such as `is_active` / `is_transformed`, render frame sequences with a green bounding box drawn, feed them to Gemini, and generate an **object-centric caption + summary**. Output: `summary_{train,val}_{min_clip}s_{min_video}m.json`.

3. **Human annotation & rating**
   Human review and correction of the generated captions (`video_caption_rating_2025xxxx.csv`). Inappropriate ones are removed and overwritten with corrected captions.

4. **Benchmark construction** — `dataset/build_benchmark.py`
   Match the rating CSV (`Completed` rows only) against the original EgoTracks boxes, flatten each occurrence, and output `longegorefer_wo_interaction.json` (the version without `interaction`).

5. **Interaction extraction** — `dataset/extract_interaction.py`
   Extract human–object interaction verb phrases from each `caption` using GPT, add the `interaction` field, and output this file `longegorefer.json`.

Helper scripts:
- `dataset/check_dataset.py` — Statistics and histograms of clip length, video length, and caption word count.
- `dataset/post_process.py` — Write out `caption.txt` / `time.txt` for annotation.
- `dataset/save_clip_with_bbox.py`, `dataset/save_video_with_bbox.py` — Save box-annotated videos (for quality checking).

## Tasks & evaluation

Baseline and agent implementations live under `src/`. They take a referring expression (`caption`) as input and localize the target object within a long video.

### Temporal Grounding
Given a video and a referring expression, predict the **time interval** in which the target object behaves as described.

- Baseline: `src/temporal_grounding_baseline.py` (feeds video + expression to Gemini)
- GPT variants & ablations: `src/temporal_grounding_baseline_gpt*.py`
- Metrics: **mIoU**, **Recall@{0.1, 0.3, 0.5}** (ground truth is `video_*_frame_number / 30` seconds)

### Spatial Grounding
Within the predicted (or ground-truth) time interval, predict the target object's **bounding box / spatio-temporal track**.

- Baseline: `src/spatial_grounding_baseline_grounded*.py` (Gemini + Grounding DINO + SAM2)
- Metrics: **vIoU** (spatio-temporal IoU), etc. Uses `bbox` as ground truth.
